导入样式时，link和@import的区别

2024-12-24 星期二
面试题
  • link 是HTML标签，@import是css提供的
  • link 引入的样式，页面加载时同时加载；@import 引入的样式需要等页面加载完成后再加载
  • link 没有兼容性问题，@import 不兼容ie5以下
  • link 可以通过js动态引入样式表修改样式，而@import 不可以

性能优化的建议

  • 避免在css文件中使用过多的@import
  • 关键css样式直接使用link 引入
  • 使用css模块打包工具，例如webpack 合并文件
  • 对非关键样式使用异步加载或按需加载
  • 理由浏览器缓存机制优化加载性能

现代开发趋势

  • 使用构建工具处理css依赖，例如webpack vite
  • 采用css模块化方案，例如 CSS modules CSS-in-js
  • 使用预处理器管理样式依赖
