Promise.all的使用场景

2024-07-16 星期二
js
  • Promise.all 是 JavaScript 中用于处理多个并发异步操作的一个方法。它接受一个包含多个 promise 的可迭代对象（通常是一个数组），并返回一个新的 promise。当传入的所有 promise 都解决（resolve）时，新返回的 promise 才会解决。如果其中一个 promise 拒绝（reject），则新返回的 promise 也会立即拒绝，并返回第一个拒绝的原因。

使用场景

  1. 并发多个异步操作：

    • 当需要同时发起多个异步请求，并等待所有请求完成后再进行后续处理时。例如，获取多个 API 数据，然后在获取所有数据后进行处理。

  2. 数据聚合：

    • 当需要从不同的数据源获取数据，并在所有数据获取完成后进行汇总或合并操作时。

  3. 加载依赖资源：

    • 当需要加载多个依赖资源（如图像、脚本、样式表等），并在所有资源加载完成后初始化页面或应用程序时。

注意事项

  • 如果传入的 promise 中有一个失败（reject），Promise.all 返回的 promise 会立即失败，而不会等待其他 promise 的结果。
  • 如果需要在一个 promise 失败后仍然获取其他 promise 的结果，可以使用 Promise.allSettled，它会等待所有 promise 都完成，不管它们是解决还是拒绝。
