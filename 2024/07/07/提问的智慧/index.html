提问的智慧

2024-07-07 星期日
转载

原文

在提问之前

  • 尝试在你准备提问的论坛的旧文章中搜索答案。
  • 尝试上网搜索以找到答案。
  • 尝试阅读manual以找到答案。
  • 尝试阅读常见问题文件（FAQ）以找到答案。
  • 尝试自己检查或实验以找到答案。
  • 向你身边的强者朋友打听以找到答案。
  • 如果你是程序开发者，请尝试阅读源代码以找到答案。

在你提问时

  • 慎选提问的论坛
  • Stack Overflow
  • 使用有意义且描述明确的标题
  • 精确的描述问题
  • 话不在多在精
  • 低声下气不能代替你的功课
  • 描述问题而非你的猜测
  • 描述目标而不是过程
  • 清楚明确的表达你的问题和需求
  • 别把自己的家庭作业问题贴上来
  • 礼貌
  • 问题解决后，加个简短的补充说明

如何解读答案

  • read the fucking manual 去读文档
  • search the fucking web 谷歌一下

如何更好的回答问题

  • 态度和善一点
  • 对新手多给予包容
  • 如果你不确定，一定要说出来
  • 如果帮不了忙，也别妨碍他
  • 试探性的反问可以引出更多的细节
